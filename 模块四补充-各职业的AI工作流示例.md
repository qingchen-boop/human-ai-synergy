# 模块四补充：各职业的AI工作流示例

> 本文档提供不同职业场景下的具体AI工作流，帮助学习者将模块四的理念落地到实际工作中。

---

## 📋 如何使用本文档

每个职业的AI工作流包含：
- **核心工作场景**：该职业最耗时或最需要AI辅助的任务
- **推荐工具类型**：适合该职业的AI工具类别
- **具体工作流**：可立即实践的步骤
- **避坑提示**：该职业使用AI时最容易犯的错误

> 提示：工具名称仅为示例，实际选择时请根据可获取性和个人偏好决定。

## 🏥 医疗健康从业者

### 核心场景
- 医学文献检索与综述
- 病历整理与初步分析
- 患者教育材料撰写
- 继续教育学习

### AI工作流

#### 文献研究工作流
```
1. 确定研究问题（你）
   → "我想了解糖尿病合并肾病患者使用SGLT2抑制剂的最新证据"
   
2. AI协助设计检索策略（AI+你）
   → "帮我列出这个主题的关键词组合，以及可能的相关MeSH术语"
   
3. 执行检索，获取文献（你）
   
4. AI辅助初步筛选（AI）
   → 将文献摘要发给AI："请判断这些研究是否涉及SGLT2抑制剂在糖尿病肾病中的应用，列出可能相关文章"
   
5. AI协助撰写综述框架（AI）
   → "根据我的文献笔记，帮我整理成综述的大纲"
   
6. 你撰写核心观点和结论（你）
```

#### 患者沟通工作流
```
1. 整理患者病历信息（你）
   
2. AI协助准备解释材料（AI）
   → "我用通俗语言解释一下这个检查结果，并列出患者可能想问的5个问题"
   
3. 你审核并补充（你）
   
4. 发送给患者前最后确认（你）
```

### ⚠️ 医疗AI使用边界
```
❌ 绝对不能用AI：
• 独立做出诊断决策
• 开具处方或调整药物剂量
• 替代临床判断

✅ 可以用AI：
• 文献检索和初步整理
• 撰写患者教育材料（需审核）
• 继续教育的学习辅助
• 病历结构化整理
```

### 适用工具类型
- 文献管理工具（整合AI摘要功能）
- 医学知识库（UpToDate等+AI增强）
- 语音转写工具（门诊记录整理）

## ⚖️ 法律从业者

### 核心场景
- 法律文书草稿
- 案例研究
- 合同审查
- 法律研究

### AI工作流

#### 合同审查工作流
```
1. 获取合同文本（你）
   
2. AI初步审查（AI）
   → "请识别这份合同中的关键条款，特别关注：
     • 责任限制条款
     • 违约金条款  
     • 终止条件
     • 争议解决条款"
   
3. AI列出潜在风险点（AI）
   → "基于上述分析，列出这份合同对我方可能存在的3个最大风险"
   
4. 你进行专业判断（你）
   
5. AI协助撰写修改建议（AI）
   → "将我的修改意见整理成正式的合同修订建议函"
```

#### 法律研究工作流
```
1. 确定法律问题（你）
   → "某电商平台用户协议中的仲裁条款是否具有法律效力？"
   
2. AI协助确定研究框架（AI+你）
   → "关于这个问题，需要考虑哪些法律维度？请列出分析框架"
   
3. 针对性检索法律条文和案例（你）
   
4. AI辅助分析（AI）
   → "根据我提供的案例摘要，分析法院通常如何认定这类条款的效力"
   
5. 你撰写分析意见（你）
```

### ⚠️ 法律AI使用边界
```
❌ 绝对不能用AI：
• 独立提供法律意见
• 用AI生成的文书直接提交（必须经过律师审核）
• 依赖AI对法律条文的解释做最终判断

✅ 可以用AI：
• 文书初稿整理
• 案例摘要整理
• 合同初步审查（仅作为人工审核的辅助）
• 法律研究的方向指引
```

## 💼 企业管理者

### 核心场景
- 会议纪要整理
- 报告撰写
- 数据分析
- 战略规划

### AI工作流

#### 周会高效工作流
```
1. 会议中使用语音转写（你/工具）
   
2. AI转录整理（AI）
   → 将录音转为文字稿
   
3. AI生成结构化纪要（AI）
   → "请将这份会议记录整理成结构化纪要，包含：
     • 决策事项（需要明确的内容）
     • 行动项（负责人+截止日期）
     • 待跟进问题
     • 讨论要点摘要"
   
4. 你审核并补充关键判断（你）
   
5. 分发纪要（你）
```

#### 报告撰写工作流
```
1. 明确报告目的和受众（你）
   
2. AI协助收集素材（AI）
   → "帮我整理Q3运营数据的要点，并列出需要补充的数据缺口"
   
3. 你确定报告核心观点（你）
   
4. AI生成初稿框架（AI）
   → "根据这些数据，帮我设计报告的结构"
   
5. 你填充关键洞察和判断（你）
   
6. AI优化表达和格式（AI）
   → "帮我把这部分内容改得更简洁有力"
```

### ⚠️ 管理AI使用边界
```
❌ 绝对不能用AI：
• 涉及员工绩效评估的最终判断
• 机密商业决策的独立分析
• 替代人际沟通的场景（如裁员谈话）

✅ 可以用AI：
• 会议纪要整理
• 数据报告初稿
• 商业文档的语言优化
• 头脑风暴的思路扩展
```

## ✍️ 内容创作者

### 核心场景
- 选题头脑风暴
- 初稿撰写
- 内容改写
- 素材收集

### AI工作流

#### 视频创作完整工作流
```
阶段1：选题策划
1. AI协助选题（AI+你）
   → "我是做科技产品评测的，请列出最近10个可能爆款的选题"
   
2. 你评估可行性（你）
   
阶段2：脚本撰写
3. AI生成脚本初稿（AI）
   → "帮我写一个关于[选题]的5分钟视频脚本，包含开场、核心内容、结尾"
   
4. 你改写核心观点和个人风格（你）
   
5. AI协助细节打磨（AI）
   → "帮我把这段话改得更口语化，更有感染力"
   
阶段3：后期优化
6. AI协助生成字幕（工具）
   
7. AI辅助标题优化（AI）
   → "提供10个吸引人的视频标题选项"
```

#### 文章创作工作流
```
1. 收集素材和灵感（你+AI辅助）
   → "帮我整理关于这个主题的正反方观点"
   
2. 确定文章核心论点（你）
   
3. AI生成文章大纲（AI）
   → "基于这个主题，生成一个800字文章的大纲"
   
4. 你撰写核心段落（你）
   
5. AI协助填充辅助内容（AI）
   → "帮我补充这个观点的案例和数据支撑"
   
6. AI润色全文（AI）
   
7. 你最终审核（确保无事实错误）
```

### ⚠️ 内容创作AI使用边界
```
❌ 绝对不能用AI：
• 直接提交AI生成的整篇内容（需标注或大量修改）
• 用AI虚构个人经历或故事
• 未经审核使用AI提供的事实性陈述

✅ 可以用AI：
• 头脑风暴扩展思路
• 初稿打破空白页恐惧
• 语言润色和改写
• 标题和描述优化
```

## 📊 数据分析师

### 核心场景
- 数据清洗
- 代码编写
- 报告生成
- 趋势分析

### AI工作流

#### 日常分析工作流
```
1. 明确分析目标（你）
   
2. AI协助生成代码（AI）
   → "用Python帮我写一段代码，读取这个CSV文件，清洗缺失值，并做数据可视化"
   
3. 你审核并调整代码（你）
   
4. AI协助结果解读（AI）
   → "帮我解读这个散点图的分布特征，以及可能的业务含义"
   
5. AI协助撰写分析报告（AI）
   → "基于这个分析结果，帮我写一份简短的报告摘要"
```

#### SQL查询工作流
```
1. 描述需求（你）
   → "我需要从orders表中找出每个用户最近一个月的订单总金额"
   
2. AI生成SQL（AI）
   
3. 你审核逻辑正确性（你）
   
4. AI协助优化（AI）
   → "这个查询运行较慢，帮我优化"
```

### ⚠️ 数据分析AI使用边界
```
❌ 绝对不能用AI：
• 未经审核直接使用AI生成的SQL/代码
• 依赖AI对数据意义的解读做业务决策
• 用AI生成的数据替代真实数据

✅ 可以用AI：
• 代码生成和debug
• 基础统计分析的自动化
• 报告初稿撰写
• 图表代码生成
```

- 教案设计
- 试题生成
- 学生反馈
- 家长沟通

### AI工作流

#### 备课工作流
```
1. 确定教学目标（你）
   
2. AI协助设计教案框架（AI）
   → "基于这个知识点，帮我设计一个2课时的教案，包含学习目标、教学活动、评估方式"
   
3. 你调整和补充教学经验（你）
   
4. AI协助制作辅助材料（AI）
   → "帮我生成5个课堂提问，引导学生思考"
   
5. 你筛选和修改（你）
```

#### 学生评语工作流
```
1. 收集学生表现信息（你）
   
2. AI协助撰写初稿（AI）
   → "根据以下信息，帮我写一段期末评语：[学生表现概述]"
   
3. 你审核并加入个人观察（你）
   
4. AI协助调整语气和长度（AI）

### ⚠️ 教育AI使用边界
```
❌ 绝对不能用AI：

✅ 可以用AI：
• 教案和教学材料辅助生成
• 试题生成和审核

• 家长沟通模板辅助

### 核心场景
- 代码编写
- 代码审查
- 文档撰写
- 问题调试

### AI工作流

#### 日常开发工作流
```
1. 理解需求（你）
   
2. AI协助设计解决方案（AI）
   → "帮我设计一个用户认证模块的技术方案"
   
3. 你选择方案并设计细节（你）
   
4. AI生成代码（AI）
   → "帮我生成这个功能的Python代码"
   
5. 你审核代码（你）
   
6. AI协助写测试用例（AI）
   → "帮我为这个函数生成单元测试"
   
7. AI协助调试错误（AI）
   → 粘贴错误信息 → "这个错误可能的原因是什么？"
```

#### 代码审查工作流
```
1. 你完成代码编写
   
2. AI协助初步审查（AI）
   → "请审查这段代码的潜在问题，包括：
     • 安全性
     • 性能
     • 可读性
     • 边界条件处理"
     
3. 你做最终审查决策（你）
```

```
❌ 绝对不能用AI：
• 直接使用AI代码而不理解原理
• 用AI生成的代码直接部署到生产环境
• 依赖AI替代自己的技术判断

✅ 可以用AI：
• 代码补全和生成
• Debug辅助
• 代码解释
• 文档辅助撰写
• 学习新技术时的问答
```

## 🏠 通用职场人（万能工作流）

### 信息处理工作流（适合所有人）
```
每日信息消化：
1. 收集信息（邮件、文档、新闻等）（你）
   
2. AI快速分类和总结（AI）
   → "帮我分类这些未读邮件，标注优先级"
   
3. 你做关键决策（你）
   
4. 需要深挖的 → AI协助分析（AI）

```
1. 发起会议前：AI协助准备议程（AI）
   → "帮我设计一个1小时的头脑风暴会议议程"

3. 会议后：AI生成纪要和待办（AI）

1. 批量收集待回复消息（你）

## 📝 如何设计自己的AI工作流

### 5步法（来自模块四）

```
Step 1: 盘点高频任务
→ 列出你每天/每周重复做的任务

Step 2: 识别AI切入点
→ 哪些任务有明确的输入输出？
→ 哪些任务需要创意 vs 只需要执行？

Step 3: 设计触发条件
→ 什么时候用AI？每次？还是只在特定情况？

Step 4: 明确AI角色
→ AI是：头脑风暴伙伴？初稿生成器？审核工具？

Step 5: 建立反馈循环
→ AI做得好的地方？
→ AI最容易出错的地方？
→ 如何改进？
```

### 自查问题清单

```
□ 我用AI做什么？（明确边界）
□ 我不用AI做什么？（划定红线）
□ 我怎么确保AI输出经过我的审核？
□ 我如何避免对AI形成依赖？
□ 我的独特价值在哪里，AI无法替代？
```
