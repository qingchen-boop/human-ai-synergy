## 第一课：提示工程基础

### 什么是提示工程？

不是"学会怎么问AI"，而是"学会怎么设计人机对话的系统"。

**三个层次：**

```
Level 1: 会问问题 → 问什么答什么
Level 2: 会结构化提问 → 得到更精准的答案  
Level 3: 会设计对话系统 → 引导AI产出你想要的结果
```

### 提示的四大要素

#### 1. 角色设定（Role）

```
❌ 不好："帮我写一段话"

✅ 好："你是一个资深科技记者，用简洁有力的语言
       写一段关于AI最新进展的新闻导语"
```

**为什么有用：** AI会调用与该角色相关的知识模式和语言风格

#### 2. 任务描述（Task）

- 具体 > 模糊
- 结果导向 > 过程导向

```
❌ 模糊："帮我看看这篇文章"

✅ 具体："指出这篇文章的三个逻辑漏洞，
         每个漏洞附上具体例子"
```

#### 3. 约束条件（Constraints）

```
✅ "字数限制在200字以内"
✅ "用第二人称写作"
✅ "避免使用专业术语"
✅ "格式：用要点列表，不是段落"
```

#### 4. 上下文（Context）

给AI足够的信息，让它理解你的处境。

```
❌ 少："推荐一本书"

✅ 多："我想读一本关于认知科学入门的书，
       目的是了解人类决策的常见误区，
       偏好轻松易读、有趣案例多的风格"
```

|----------|--------|

## 第二课：任务分解与人机分配

### 核心原则

**不是"人做"或"AI做"，而是"人+AI怎么做"**

### 分解策略

**什么时候该分解：**
- 任务太大，AI输出容易失控
- 混合了AI擅长和不适配的部分
- 需要迭代优化，不能一步到位

**分解示例：写一篇产品分析报告**

```
第一步（人）：确定分析维度 → AI无法代替你思考你要分析什么
第二步（AI）：搜集行业背景 → AI可以快速总结已知信息
第三步（人）：补充你的第一手体验 → 这是AI没有的
第四步（AI）：生成报告初稿 → AI擅长文本生成
第五步（人）：审核、修改、补充 → 人做最终把关
第六步（AI）：润色、排版 → AI擅长
```

### 任务分配决策树

```
任务开始
    │
    ▼
这个任务的核心价值是什么？
    │
    ├─ 情绪/审美/价值观 → 人类做主
    │
    ├─ 信息搜集/格式转换 → AI做主
    │
    ├─ 创意发散 → AI先做，人筛选
    │
    ├─ 精确事实/实时信息 → 查资料（人或搜索）
    │
    └─ 风险性决策 → 人为主，AI为辅
```

## 第三课：人机迭代回环

### 什么是迭代回环？

人机协作不是"一问一答"，而是持续对话、逐步逼近目标。

```
┌─────────────────────────────────────┐
│  你的目标                            │
└─────────────────────────────────────┘
              ↑ 反馈
              ↓ 调整
┌─────────────────────────────────────┐
│  AI输出 ← 你告诉它哪里不对          │
└─────────────────────────────────────┘
              ↑ 再生成
              ↓
         （循环直到满意）
```

### 有效迭代的技巧

#### 1. 给出具体反馈，而不是笼统评价

```
❌ "不对，不是我想要的"
✅ "这个方向对了，但第二点太学术，
    换成普通人能懂的表述"
```

#### 2. 利用AI的"自我修正"能力

```
"你的上一个回答提到了X，但我指的是Y，
请重新回答Z，同时避免X"
```

#### 3. 控制迭代次数

- **简单任务：** 1-2次迭代
- **中等任务：** 3-5次迭代
- **复杂任务：** 设立"迭代上限"，防止无限循环

### 案例：迭代写一封道歉邮件

```
第1次：AI写一封道歉邮件
反馈："语气太正式了，我想要更真诚的感觉"
第2次：调整语气
反馈："好多了，但第三段太长了，删掉"
第3次：完成
```

## 第四课：多AI协作

### 为什么用多个AI？

不同AI有不同的特长，就像不同的人有不同的技能。

| AI | 擅长 | 不擅长 |
|----|------|--------|
| 擅长逻辑推理的AI | 分析、规划 | 创意发散 |
| 擅长创意的AI | 头脑风暴 | 结构化输出 |
| 擅长精准事实的AI | 查事实 | 流畅写作 |

### 多AI协作模式

**模式A：串行接力**
```
AI-A输出 → AI-B优化 → AI-C检查
例：ChatGPT写初稿 → Claude改结构 → GPT再润色
```

**模式B：并行对比**
```
同时问多个AI同一个问题，比较答案
注意：不是选"最优"，而是发现差异
```

**模式C：专门分工**
```
这个AI负责查资料 → 那个AI负责写作 → 另一个负责校对
```

## 第五课：质量校验

### 为什么校验？

AI输出可能是错的，你必须成为质量的最终把关者。

### 校验清单

```
□ 事实核查：AI提到的具体事实是否可验证？
□ 逻辑检查：推理过程有没有漏洞？
□ 一致性：前后表述是否矛盾？
□ 适用性：答案适合我的具体情况吗？
□ 完整性：是否遗漏了重要的方面？
```

### 实用校验方法

1. **反向提问：** "请用相反观点回答同样的问题"
2. **要求证据：** "这个结论的依据是什么？"
3. **简化测试：** "用一句话总结你的回答"
4. **换AI验证：** 用另一个AI检查同一个问题
