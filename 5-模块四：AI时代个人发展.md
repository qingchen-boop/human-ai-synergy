## 第一课：用AI学习

### 传统学习 vs AI辅助学习

| 传统学习 | AI辅助学习 |
|----------|------------|
| 遇到不懂的 → 查书/搜索 → 可能找不到答案 | 随时提问，获得即时解释 |
| 被动接收知识 | 可以苏格拉底式追问 |
| 只能自己思考 | 可以让AI扮演不同观点 |
| 学习进度一致 | 可以按自己的节奏来 |

### 正确用法

#### 1. 苏格拉底式提问

不要让AI给你答案，让AI帮你思考。

```
❌ "告诉我什么是量子力学"

✅ "我理解量子力学是关于微观粒子的理论，
    但我不明白为什么观测会影响结果。
    请用生活化的例子帮我理解这个矛盾"
```

```
✅ "你认为这个观点是对的吗？有什么可能的反驳？"
✅ "如果这个理论是错的，哪里最可能出错？"
✅ "有没有其他学科的知识可以帮助理解这个问题？"
```

#### 2. 知识解析

让AI用**你的水平**解释概念：

```
✅ "用初中数学水平的语言解释微积分"
✅ "我是一个程序员，用编程类比帮我理解这个概念"
✅ "用比喻的方式解释，不要用任何专业术语"
```

#### 3. 学习监督

让AI监督你的学习进度：

```
"我计划一个月学完这本教材，
请每周问我三个关键问题来测试我的理解"
```

### 错误用法（避免！）

```
❌ 让AI替你学习（你不动脑，AI也无法真正教会你）
❌ 完全依赖AI解释（AI可能错，你无法判断）
❌ 用AI记笔记代替自己思考（思考过程才是学习）
❌ 把AI的答案当作真理（保持批判）
```

## 第二课：用AI创作

### 创作的AI增强框架

```
           ┌──────────────┐
           │ 你的独特视角  │
           └──────┬───────┘
                  ↓
┌─────────────────────────────────────────┐
│            AI可以帮你做：                │
│  • 头脑风暴，扩展思路                    │
│  • 初稿生成，打破空白页恐惧              │
│  • 格式转换（口语→书面，总结→详细）      │
│  • 语法润色，语言优化                    │
│  • 提供不同风格的版本                    │
└─────────────────────────────────────────┘
                  ↓
           ┌──────────────┐
           │ 你的独特贡献  │
           │ 核心观点      │
           │ 个人经历      │
           │ 真实情感      │
           │ 价值判断      │
           └──────────────┘
                  ↓
           ┌──────────────┐
           │ 最终作品 =    │
           │ 你的独特 + AI │
           └──────────────┘
```

### 各场景的AI使用策略

#### 写作

| AI能做的 | 你必须做的 |
|----------|------------|
| 初稿、框架、灵感 | 核心观点、故事、情感 |
| 语法、流畅度 | 真实性、独特视角 |
| 改写、润色 | 价值观判断 |

#### 设计/艺术

| AI能做的 | 你必须做的 |
|----------|------------|
| 快速生成多个方案 | 选择最符合目标的 |
| 提供视觉灵感 | 判断审美质量 |
| 素材处理、合成 | 整体创意方向 |

#### 学习/研究

| AI能做的 | 你必须做的 |
|----------|------------|
| 文献综述、总结 | 研究问题、假设 |
| 资料查找 | 判断质量、相关性 |
| 初稿写作 | 原创性贡献 |

### 创作诚信底线

```
❌ 不要：
• 直接提交AI写的整篇文章作为自己的作业/作品
• 把AI生成的内容当作自己的经历
• 不经审核使用AI的事实性声明

✅ 要：
• 清楚标注AI辅助的部分
• 对AI内容负责——你审核过，你负责
• 把AI当作工具，不是枪手
```

## 第三课：用AI决策

### AI能帮决策吗？

**能，但记住：AI不知道什么对你最重要**

### 决策辅助框架

```
问题识别（你）→ 信息收集（AI+你）→ 选项生成（AI）→ 
评估分析（AI+你）→ 风险评估（AI+你）→ 最终决定（你）
```

### 实用场景

#### 1. 利弊分析

```
✅ "我在考虑转行，请帮我分析这个决定可能的利弊，
    我现在做的是X，有Y年的经验"
```

**注意：** AI不知道你真正在乎什么，你自己补充价值判断。

#### 2. 视角补充

```
✅ "我倾向于A方案，但我想听听反对意见，
    请从不同角度指出A方案可能的问题"
```

#### 3. 风险评估

```
✅ "如果我这么做，最坏的情况是什么？
    发生的可能性有多大？我可以怎么准备？"
```

### 什么时候不该用AI决策？

```
❌ 涉及重大道德/伦理选择
❌ 需要你独特情况的信息（AI不了解你）
❌ 后果严重且不可逆（这时候你应该找真人咨询）
❌ 涉及你没有共享给AI的个人信息
```

## 第四课：个人AI工作流设计

### 什么是AI工作流？

把AI嵌入到你日常生活和工作的特定环节中，形成高效循环。

### 设计步骤

```
Step 1: 盘点你的高频任务
Step 2: 识别哪些任务适合AI辅助
Step 3: 设计"触发条件"（什么时候用AI）
Step 4: 确定AI的具体角色
Step 5: 设定输出标准（你怎么判断AI做得好不好）
Step 6: 实践、调整、优化
```

### 示例工作流

#### A. 信息消化工作流

```
阅读/看到信息 → AI总结要点 →
你判断重要性 → 需要深挖的 →
进一步问AI → 形成自己的理解
```

#### B. 写作工作流

```
想写主题 → AI帮你列提纲 →
你确认方向 → AI写初稿 →
你改写核心部分 → AI润色 →
你最后审核
```

#### C. 学习工作流

```
学到新概念 → 问AI三个问题 →
尝试用自己的话解释 → AI给反馈 →
记笔记（自己的话）→ 定期复习
```

### 建立你的AI使用原则

```
□ 我用AI做什么？□ 我不用AI做什么？
□ 我怎么确保AI输出经过我的审核？
□ 我如何避免过度依赖AI？
□ 我什么时候知道自己对AI形成了不当依赖？
```

## 第五课：AI边界——什么时候不该用AI

### 这些情况，请放下AI

| 场景 | 原因 | 替代方案 |
|------|------|----------|
| 情感支持 | AI没有真正的共情，可能让你感觉更糟 | 朋友、家人、心理咨询师 |
| 重大人生决策 | AI不了解你的完整情况，不懂你的价值观 | 真人顾问、信任的人 |
| 你想自己享受的过程 | 不用AI偷懒是为了保留人的成长体验 | 享受当下，即使更慢更难 |
| 需要身体力行的事 | 有些技能只有通过身体才能真正学会 | 亲自做 |
| 当AI成为逃避 | 用AI回避困难、痛苦、必要的挣扎 | 面对，但可以AI辅助学习 |

### 过度依赖AI的信号

```
⚠️ 你注意到自己：
• 越来越不想独立思考
• 没有AI就无法开始任何任务
• 开始分不清哪些想法是你的，哪些是AI的
• 用AI替代了真实的人际交往
• 明知道答案还是要问AI
```
