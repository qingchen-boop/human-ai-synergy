## 模块一答案

### 练习1.1：AI能力边界测试

**参考分析维度：**

```
| 适合度 | 任务类型 | 举例 |
|--------|----------|------|
| AI主导 | 信息查询/翻译/格式化 | 写邮件/翻译/总结 |
| AI辅助 | 需要判断的创意任务 | 头脑风暴/初稿/代码 |
| 人类主导 | 需要真实经验/情感/价值观 | 重大决策/安慰朋友/价值观判断 |
```

**差距分析常见发现：**
- AI在信息整理方面往往比预期好
- AI在需要"常识"的任务上容易出错
- 创意任务AI能提供灵感但质量不稳定
- 专业领域AI可能比预期更好（知识广）也可能更差（幻觉）

---

### 练习1.2：幻觉识别训练

**常见幻觉模式：**

```
容易出现幻觉的情况：
• 具体日期/数字/统计数据
• 小众人物/事件/作品
• 专业领域的边缘知识
• 涉及时间的预测

AI表现"不确定"的信号：
• "截至我的知识截止日期..."
• "我不确定..."
• "可能有..."
• 回答变得模糊

识别技巧：
✅ 问具体到无法编造的细节
✅ 多次问同一个问题看是否一致
✅ 要求AI提供信息来源
✅ 用搜索引擎交叉验证
```

### 练习1.3：任务分类实践

**参考答案：**

```
a) 写一封正式的工作邮件
   → AI主导/AI辅助
   理由：格式规范，AI擅长

b) 决定下周是否要出差
   → 人类主导
   理由：涉及你的具体情况、偏好、价值观

c) 查找2024年诺贝尔经济学奖得主
   → AI辅助（最终需核实）
   理由：事实查询，但可能不准确

d) 给焦虑的朋友一些安慰建议
   → 人类主导
   理由：需要真实情感理解和共情

e) 帮你规划下个月的日程
   → AI辅助
   理由：需要你的具体情况，AI可以优化建议

f) 帮你写一首生日祝福诗
   → AI主导
   理由：创意格式任务，AI擅长

g) 分析你公司的季度报表
   → AI辅助
   理由：需要你解读数据，但AI可以辅助分析

h) 决定买哪款手机
   → 人类主导
   理由：涉及个人偏好和价值观
```

## 模块二答案

### 练习2.1：提示优化

**优化原则对照：**

```
好提示的特征：
✅ 具体说明任务和背景
✅ 明确说明期望的格式
✅ 设定约束条件（字数/风格/受众）
✅ 说明角色（如果你想让AI扮演）
✅ 要求解释而非只给答案

优化示例：
──────────────────────────────────────

❌ "帮我写代码"
→ ✅ "用Python写一个函数，输入一个列表，
    返回其中所有偶数的和。请加上注释说明逻辑。"

❌ "解释一下量子力学"
→ ✅ "用生活化的比喻解释量子力学中的'叠加态'概念，
    不要用专业术语，目标读者是高中生。"

❌ "给我反馈"
→ ✅ "请从这三个角度评价我的文章：
    1）逻辑是否清晰
    2）论据是否充分
    3）语言是否流畅。
    每方面给1-5分，并说明理由。"
```

### 练习2.2：论文写作人机协作流程（参考）

```
人的工作：
• 选题和研究问题（必须人做）
• 研究方法和设计
• 核心论点的确定
• 结论解读和意义
• 最终审核和质量把控

AI的工作：
• 文献检索和初步整理
• 文献综述框架
• 初稿生成（你可补充内容）
• 语法检查和润色
• 格式调整

人机回环示例：
第1轮：AI帮你整理文献笔记 → 你提炼核心观点
第2轮：AI生成综述初稿 → 你审核和补充
第3轮：AI检查格式引用 → 你最终通读
```

### 练习2.4：质量校验清单

```
□ AI说的"事实"我有核实过吗？
□ AI忽略了我的哪些重要信息？
□ AI的建议是否符合我的价值观和优先级？
□ 如果AI错了，最坏的后果是什么？
□ 我是否在用AI的判断代替自己的判断？
□ AI是否在迎合我（而不说真话）？
□ 这个决定可以撤销吗？还是不可逆的？
```

### 练习3.2：权力分析框架

```
以ChatGPT为例：

1. 谁做的？
   → OpenAI（美国公司）

2. 商业模式？
   → API收费 + Plus订阅 + 企业合作

3. 你使用，他们获得？
   → 数据（对话）
   → 用户粘性
   → 可能用于训练（争议中）

4. 你的利益和他们的利益一致吗？
   → 不完全一致：他们要优化模型，你要准确答案
   → 潜在冲突：你的数据价值 vs 他们的服务价值

5. 如果他们倒闭了？
   → 你失去这个工具
   → 可能影响你依赖AI的工作流程
```

## 模块四答案

### 练习4.1：AI学习法周计划（参考）

```
周一：探索主题
→ 问AI："关于XX主题，我应该知道哪些核心概念？"
→ 记录：5个核心概念

周二：深入一个概念
→ 让AI用类比解释其中一个
→ 自己画一张思维导图

周三：检验理解
→ 让AI提问考你
→ 回答后让AI评价

周四：应用实践
→ 用AI帮你找练习题/案例
→ 自己完成，AI给反馈

周五：知识整合
→ 用AI帮你梳理知识框架
→ 你补充你自己的理解
→ 写学习总结

周末：反思
→ AI帮我想想还有哪些盲点？
→ 下周学习计划？
```

### 练习4.3：人机工作流设计检查

```
设计检查项：

□ 我是否清楚AI在流程中的具体角色？
□ 每个AI输出是否都经过我审核？
□ 我是否保留了"退出AI"的能力？
□ 流程中是否有明确的人类决策点？
□ 我是否过度依赖AI而不自知？

迭代问题：
• AI在哪一步最有帮助？
• 哪一步AI帮了倒忙？
• 哪一步我应该自己做但让AI做了？
• 下周要调整什么？
```

## 模块五答案

### 练习5.2：AGI辩论参考

**支持AGI10年内实现：**
```
• 计算能力指数增长（摩尔定律延续）
• 大模型涌现能力不断出现
• AI在特定领域已超越人类
• 投资和研究力度空前
```

**反对AGI10年内实现：**
```
• 通用能力vs专用能力的差距巨大
• 常识、推理、迁移学习仍是难题
• 身体和情感体验难以复制
• 历史上AI预测总是过于乐观
```

**最强反驳：**
```
"AGI的实现不是技术问题，是定义问题。
 如果我们能准确定义AGI，我们可能已经实现了它。
 问题是：我们对'通用智能'的理解本身就在变化。"
```

## 综合练习答案

### 人机协作方案模板（完整示例）

```
场景：准备一场重要面试

1. 我的目标：
   → 成功通过面试，获得offer

2. 我的人机分工：
   人的工作：
   • 了解公司背景和岗位要求
   • 准备个人真实经历和故事
   • 眼神/肢体语言练习
   • 最终决定穿什么/怎么表达

   AI的工作：
   • 帮我列出常见面试问题
   • 帮我模拟练习和反馈
   • 帮我优化简历/自我介绍
   • 帮我研究公司

3. 人机回环设计：
   第1轮：AI帮我分析岗位JD → 我确认是否匹配我的经历
   第2轮：AI模拟面试 → 我回答 → AI给反馈 → 我改进
   第3轮：我自己彩排 → 找真人练习

4. 质量校验方式：
   • AI的反馈我是否认可？
   • 回答是否像"我"而不是背稿？
   • 是否有我真正不理解的问题？

5. 我作为人类必须把控的：
   • 我真正想不想去这家公司
   • 我的底线（薪资/职位）
   • 我是否愿意长期干这份工作

6. 风险点：
   • AI教的"标准答案"可能显得太套路
   • 我过度依赖AI准备，失去随机应变能力
   • 忽视了对公司文化的真实感受

7. 最终验收标准：
   • 能流利回答"你为什么想来这里"
   • 有一个让面试官印象深刻的故事
   • 问出有价值的问题
   • 结束后感觉：我发挥出了真实水平
```

## 反思记录示例

```
日期：2026-xx-xx
练习名称：练习4.2 创作放大

1. 我学到了什么？
   → AI能快速生成初稿，但核心洞察必须是我自己的
   → "优化"比"生成"更适合把AI当工具

2. 什么最难？
   → 区分哪些想法是AI给我的，哪些是我自己的
   → 不被AI的"流畅表达"带跑

3. 什么和我想的不一样？
   → AI写的东西看起来很好，但深度不够
   → 我需要提供更多真实经历，AI才能帮我放大

4. 我接下来要做什么？
   → 下次创作前先自己列提纲
   → AI只用于扩展和润色，不用于构思
```
