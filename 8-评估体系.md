# 人机共智：AI时代的协作思维与实践 - 评估体系

---

## 评估理念

**不是考核你知道什么，而是考核你能做什么**

评估的核心是检验学生是否真正掌握了人机协作的思维方式和实践能力。

---

## 评估结构

| 类型 | 占比 | 说明 |
|------|------|------|
| 过程评估 | 60% | 持续性的学习记录和实践 |
| 综合考核 | 40% | 期末设计人机协作方案 |

---

## 过程评估（60%）

### 1. 每周人机协作日志（30%）

**要求：**
- 每周至少提交2篇日志
- 记录你真实使用AI的经历

**格式：**
```
日期：2026-xx-xx
任务背景：________________
我的目标：________________
我用了什么AI？做了什么？
AI表现如何？给我带来了什么？
我的反思/学到什么？
```

**评分标准：**
- 真实性（不是编造的）
- 反思深度
- 与课程概念的连接

---

### 2. 个人AI工作流展示（20%）

**要求：**
- 设计并实际运行一个个人AI工作流
- 提交流程图 + 运行记录 + 效果分析

**提交内容：**
```
1. 流程图（什么人做什么AI做什么）
2. 运行记录（实际使用的情况）
3. 效果评估（AI帮我节省了多少时间/提升了什么）
4. 反思（流程的不足和优化方案）
```

**评分标准：**
- 完整性
- 实用性
- 反思性

---

### 3. 模块反思报告（10%）

**要求：**
- 每个模块提交一份简短反思（300-500字）
- 重点：你对模块核心概念的重新理解

**评分标准：**
- 与课程内容的连接
- 个人见解的深度

---

## 综合考核（40%）

### 人机协作方案设计

**任务：**
选择一个真实场景，设计一个完整的人机协作方案，并实际运行部分内容。

**可选场景：**
- 准备一场重要面试
- 写一篇深度分析文章
- 做一个重大人生决定
- 规划一次复杂项目
- 学习一个新领域的知识
- 其他（需批准）

**提交物：**

#### 方案文档（60%）

```
# 人机协作方案设计

## 1. 场景描述（500字以内）
- 背景
- 目标
- 约束条件

## 2. 人机分工设计
- 人的角色和任务
- AI的角色和任务
- 为什么这样分工？

## 3. 详细流程
- 步骤1-10
- 每一步：谁做？AI做什么？产出什么？

## 4. 质量保障
- 如何确保AI输出质量？
- 你如何审核和把关？

## 5. 风险评估
- 可能的失败点？
- 如何应对？

## 6. 实际运行记录（至少运行50%的内容）
- 实际发生了什么？
- 和预期有什么差异？
```

#### 演示/答辩（40%）

- 10分钟presentation
- 5分钟问答
- 重点展示：你的设计思路 + 实际效果

**评分标准：**

| 维度 | 标准 |
|------|------|
| 完整性 | 方案是否覆盖了所有必要环节 |
| 合理性 | 分工和流程是否经得起推敲 |
| 创新性 | 是否有独特的设计思路 |
| 可执行性 | 方案是否真的可以运行 |
| 反思深度 | 能否识别方案的问题和局限 |

---

## 评分等级

| 等级 | 分数 | 描述 |
|------|------|------|
| A | 90-100 | 优秀 - 超出预期，有独特见解 |
| B | 80-89 | 良好 - 达到要求，有一定深度 |
| C | 70-79 | 中等 - 符合基本要求 |
| D | 60-69 | 及格 - 基本完成，但有不足 |
| F | <60 | 不及格 - 未达到基本要求 |

## 评估诚信

```
核心原则：
• 你可以对AI方案负责，不能对AI方案免责
• AI可以帮你做，但最终作品代表你的判断
• 引用AI内容需要标注，但标注不等于免责

禁止：
❌ 提交完全由AI生成、未经你任何思考和修改的内容
❌ 提交你完全不理解的人机协作方案
❌ 谎报实际运行情况

允许：
✅ 用AI帮助你规划、起草、润色
✅ 让AI做你不擅长的部分
✅ 参考AI的建议做决定
```
