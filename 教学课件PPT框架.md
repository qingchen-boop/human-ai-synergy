## 第二部分：模块一（建议6-8页）

### 第4页：什么是AI？

```
标题：AI到底是什么？

内容：
• 定义：人工智能——机器模拟人类智能
• 分类：
  - 弱AI（Narrow AI）：专注单一任务
  - 强AI（General AI）：通用智能（尚未实现）

常见AI类型：
┌─────────────┬─────────────┬─────────────┐
│  机器学习    │  深度学习    │  大语言模型  │
│  从数据中    │  神经网络    │  理解生成    │
│  学习规律    │  自动特征    │  自然语言    │
└─────────────┴─────────────┴─────────────┘

类比：AI就像一个"超级学习者"
• 看过海量例子
• 能发现规律
• 但不懂道理
```

### 第5页：AI能做什么？

```
标题：AI的能力边界

内容（可视化：能力图谱）

AI擅长的：
✅ 数据处理与模式识别
✅ 快速检索与信息整合
✅ 格式转换与内容改写
✅ 模式化任务执行
✅ 多语言翻译

AI不擅长的：
❌ 真正的理解与常识
❌ 跨领域创新
❌ 情感理解与共情
❌ 模糊情境判断
❌ 价值判断与道德推理

关键点：AI表现出的"智能" ≠ 真正的理解
```

### 第6页：AI的工作原理

```
标题：AI是如何工作的？（简略版）

以大语言模型为例：

输入 → 处理 → 输出

"请帮我写一封请假邮件"

Step 1: 理解输入
   把文字转换成数字（向量）

Step 2: 生成回答
   基于学习到的语言规律
   预测下一个词

Step 3: 输出结果
   把数字转换回文字

类比：
就像一个读过图书馆所有书的人
能模仿各种写作风格
但并不理解内容的真正含义
```

### 第7页：AI的局限性

```
标题：AI的"致命"弱点

案例分析：

案例1：幻觉问题
   问AI："秦始皇的统一六国是哪一年？"
   AI答："公元前221年" ✓（碰巧对了）
   问AI："秦始皇是哪年来中国的？"
   AI答："公元前260年" ✗（完全错误）

案例2：推理错误
   问AI："如果所有的猫都是动物，
          所有的动物都会死，
          那么猫会死吗？"
   AI可能答错 ✗

核心结论：
┌────────────────────────────────────┐
│  AI很擅长"听起来对"              │
│  但不保证"真的是对"              │
│  必须保持批判性思维！              │
└────────────────────────────────────┘
```

## 第三部分：模块二（建议8-10页）

### 第9页：什么是人机协作？

```
标题：从"使用工具"到"与AI协作"

演进过程：
1. 工具时代：人类独立完成任务
2. 辅助时代：人类使用传统工具
3. 协作时代：人类+AI共同完成
4. 增强时代：AI增强人类能力

人机协作的特点：
• 不是人类做一点，AI做一点
• 而是人类负责决策，AI负责执行
• 形成1+1>2的效果

公式：
人类独特能力 + AI处理能力 = 最佳结果
判断力、创造力、共情  +  速度、准确、规模化
```

### 第10页：提示工程基础

```
标题：与AI沟通的艺术——提示工程

什么是提示（Prompt）？
   你对AI说的每一句话

为什么提示很重要？
   同样的AI，不同的提示，效果天差地别

提示工程三要素：

┌─────────────────────────────────────┐
│  1. 角色     │  给AI一个身份        │
│  2. 任务     │  清楚说明要什么      │
│  3. 格式     │  指定输出形式        │
└─────────────────────────────────────┘

示例对比：

❌ 弱提示：
   "帮我写一封邮件"

✅ 强提示：
   "你是一个商务沟通专家。
    帮我写一封英文邮件给客户，
    内容是推迟交付日期。
    语气要专业但友好，不要太生硬。
    控制在200字以内。"
```

### 第11页：提示工程进阶

```
标题：进阶技巧

技巧1：Few-shot 示例
   "以下是三封好的道歉邮件示例：
    [例1]
    [例2]
    [例3]
    请按这个风格写一封新的"

技巧2：思维链
   "请一步步思考这个问题，
    先分析原因，再给出建议"

技巧3：迭代优化
   问 → 看结果 → 指出问题 → 追问
   "这个方案不错，但成本太高，
    请重新设计一个更经济的版本"

技巧4：限制条件
   "请用小学生能听懂的话解释"
   "不要使用任何专业术语"
```

### 第12页：任务分解

```
标题：复杂任务如何处理？

核心思想：
不要让AI一次性完成复杂任务
→ 拆分成小步骤

示例：写一篇市场报告

Step 1: 让AI收集行业信息
        "帮我整理近三年电商行业的发展趋势"

Step 2: 让AI分析竞争对手
        "基于这些信息，分析三大竞争对手的优劣势"

Step 3: 你确定报告框架
        根据AI的分析，确定核心观点

Step 4: 让AI写初稿
        "基于以上分析，帮我写报告框架"

Step 5: 你修改核心内容
        加入独特洞察

Step 6: 让AI润色
        "帮我优化语言表达"

关键：人类负责"判断"，AI负责"执行"
```

### 第13页：反馈与迭代

```
标题：与AI共创的艺术

误解：
❌ AI第一版就是最终版
❌ 一次对话解决所有问题

正确：
✅ AI生成 → 你反馈 → AI调整 → 循环迭代

迭代沟通示例：

第一轮：
你："帮我写一个产品介绍"
AI：生成初稿

第二轮：
你："第二段太技术化了，能更口语化一些吗？"
AI：修改

第三轮：
你："很好，但开头不够吸引人，
    请用一个具体用户故事开头"
AI：再次修改

...

第N轮，直到满意
```

### 第15页：AI也会"偏见"

```
标题：当AI变得不公平

问题引入：
AI输出的结果，反映的是训练数据中的偏见

   某公司用AI筛选简历
   AI系统性地降低女性简历的评分
   原因：训练数据来自过往男性主导的简历

案例2：面部识别的种族问题
   早期面部识别系统对深肤色人群
   识别准确率显著较低

案例3：翻译软件的偏见
   "He is a doctor, she is a nurse"
   AI强化了性别刻板印象

核心观点：
┌────────────────────────────────────┐
│  AI的偏见，本质上是人的偏见的      │
│  规模化呈现                        │
└────────────────────────────────────┘
```

### 第17页：AI伦理原则

```
标题：AI时代需要什么原则？

四大原则：

1. 有益（Beneficence）
   AI应该增进人类福祉
   而非伤害人类

2. 无害（Non-maleficence）
   "首先，不要伤害"
   避免AI造成伤害

3. 自主（Autonomy）
   人类保持决策自主权
   不被AI完全控制

4. 公正（Justice）
   AI的利益和风险
   公平分配

讨论问题：
在日常生活中，你遇到过AI带来的伦理困境吗？
```

### 第19页：如何用AI学习？

```
标题：AI作为学习伙伴

对比：传统学习 vs AI辅助学习

传统：
看书 → 遇到不懂 → 查字典/搜索 → 可能找不到答案

AI辅助：

❌ "告诉我答案"
✅ "我理解对吗？有什么可能的反驳？"

关键：用AI帮助你思考，不是替你思考
```

### 第24页：AI Agent

```
标题：AI Agent——下一波浪潮

什么是AI Agent？
   能自主行动、完成多步骤任务的AI系统

对比：

传统AI：
   你 → 发指令 → AI → 等待 → AI执行一步 → 你再发指令

Agent AI：
   你 → 发任务 → AI自主规划 → 执行 → 反馈 → 调整 → 直到完成

示例场景：
"帮我安排一次公司团建"
→ Agent自动查日历、搜地点、比价格、发邀请

潜在风险：
⚠️ 自主性越高，控制越难
⚠️ 错误可能放大
⚠️ 责任归属模糊
```

## PPT制作建议

### 工具推荐

```
专业级：
• Keynote（Mac）
• PowerPoint（Windows）
• Google Slides（在线）

设计工具：
• Canva（模板丰富）
• Figma（协作设计）

代码生成（可选）：
• Marp（Markdown转PPT）
• reveal.js（网页PPT）
```

### 制作流程

```
1. 准备内容大纲（本框架）
   ↓
2. 选择模板/设计风格
   ↓
3. 填充内容（文字+图表）
   ↓
4. 添加动画和过渡（适度）
   ↓
5. 校对文字和逻辑
   ↓
6. 练习演讲
```

### 注意事项

```
✅ 要：
• 文字简洁，多用图表
• 留白，不要太满
• 统一的视觉风格
• 适当的图片/图标

❌ 不要：
• 大段文字堆砌
• 花哨动画过多
• 字体大小不一
• 颜色过于杂乱
```
