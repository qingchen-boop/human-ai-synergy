## 第一课：数据偏见

### 什么是数据偏见？

AI从数据中学习，如果数据本身有偏见，AI就会延续甚至放大这些偏见。

**偏见不是"AI学坏了"，而是"数据本来就这样"**

### 常见偏见类型

#### 1. 历史偏见（Historical Bias）

**来源：** 训练数据反映的是过去和现在的社会现状，而过去的社会本身就不平等。

**例子：**
- 招聘AI看到过去几十年男性更多担任高管→认为"男性更适合高管"
- 搜索"护士"出现更多女性图片→强化"护士是女性工作"的刻板印象

#### 2. 表征偏见（Representation Bias）

**来源：** 某些群体在训练数据中被代表不足。

**例子：**
- 人脸识别对深肤色人群准确率更低（训练数据以浅肤色为主）
- 语音识别对非标准口音识别率低
- 医疗AI对女性和少数族裔的诊断准确率更低

#### 3. 测量偏见（Measurement Bias）

**来源：** 用什么指标来"测量"一个东西本身就有偏向。

**例子：**
- 用"在线广告点击率"衡量用户兴趣 → 更容易推送吸引特定群体的内容
- 用"信用分数"评估还款能力 → 可能歧视低收入群体

### 如何识别AI偏见？

```
识别清单：
□ AI的输出是否对某个群体有系统性不同的描述？
□ 是否有某个群体被系统性地排除在AI的服务之外？
□ AI是否在强化某种"常识"而这种常识本身有偏见？
□ 如果我是被AI负面影响的群体，我会怎么看？
```

### 应对策略

| 层面 | 行动 |
|------|------|
| 个人 | 意识到AI可能存在偏见，保持警惕 |
| 社会 | 要求AI系统进行偏见审计和透明报告 |
| 技术 | 开发更包容的数据集和公平性指标 |

## 第二课：权力结构

### 谁控制AI？

AI不是从天上掉下来的公平工具，它背后有**人**和**组织**。

### AI的权力地图

```
┌─────────────────────────────────────────────────────────┐
│                        用户                             │
└─────────────────────────────────────────────────────────┘
                          ↑ 使用
┌─────────────────────────────────────────────────────────┐
│              AI产品层（OpenAI、Google...）               │
│              产品设计、数据政策、访问权限                 │
└─────────────────────────────────────────────────────────┘
                          ↑ 调用
┌─────────────────────────────────────────────────────────┐
│              基础模型层（训练、微调）                     │
│              模型能力、限制、行为特征                     │
└─────────────────────────────────────────────────────────┘
                          ↑ 依赖
┌─────────────────────────────────────────────────────────┐
│              数据层（互联网文本、书籍、对话）             │
│              谁产生数据？谁选择数据？谁有数据？            │
└─────────────────────────────────────────────────────────┘
                          ↑ 采集
┌─────────────────────────────────────────────────────────┐
│              硬件层（GPU、数据中心）                      │
│              算力垄断、能源消耗                           │
└─────────────────────────────────────────────────────────┘
```

### 关键问题

1. **数据红利：** 几十亿人贡献了训练AI的数据，但收益归少数公司，这公平吗？
2. **API权力：** AI公司可以随时改变API价格、限制访问，用户几乎没有议价能力
3. **内容控制：** AI公司可以决定AI"应该"说什么和"不应该"说什么
4. **赢家通吃：** 训练大模型需要巨大投入，是否会形成技术寡头？

### 批判性思维练习

```
问自己：
• 这个AI产品的商业模式是什么？它从谁身上赚钱？
• 我的数据是怎么被使用的？
• 谁可以从这个AI的存在中获益？谁的权益可能受损？
• 如果这个AI消失了，社会会有什么损失？
```

## 第三课：就业冲击

### AI会抢走工作吗？

**会，也不会。**

### 受冲击的职业类型

| 职业类别 | 冲击程度 | 原因 |
|----------|----------|------|
| 数据录入、基础客服 | **高** | 规则清晰，重复性高 |
| 基础翻译、写作 | **中高** | AI质量已接近人类 |
| 初级分析、编程 | **中** | AI可以完成部分工作 |
| 需要人际关系的职业 | **较低** | AI难以替代情感连接 |
| 创意与决策 | **中等** | AI是工具，不是主人 |

### 历史的启示

每次技术革命都消灭了一些工作，也创造了一些工作：

```
工业革命：农民 → 工人
电脑革命：打字员 → 程序员
AI革命：... → ???
```

**关键区别：** 这次AI影响的不仅是体力劳动，还包括认知劳动。

### 职业人的应对策略

1. **不要和AI比效率** → 做AI做不了的
2. **学会和AI协作** → 把AI变成你的放大器
3. **培养人际技能** → AI最难替代的是情感和信任
4. **保持学习** → 技术变化快，持续适应

## 第四课：知识生产与信息生态

### AI如何影响我们知道的？

#### 问题一：AI生成内容的可靠性

```
传统知识生产：人写 → 同行评审 → 发表 → 质疑 → 修正
AI知识生产：概率生成 → ？ →  ?
```

**问题在于：** AI没有"同行评审"机制，它生成的错误信息可能看起来很权威。

#### 问题二：信息污染

- AI可以批量生成看似真实的假新闻
- 真假信息混杂，信息生态恶化
- 人类越来越难分辨信息来源的可靠性

#### 问题三：知识权威的转移

```
过去：专家/机构 → 知识
未来：AI → 知识？

那谁来决定AI输出的知识是对的呢？
```

### 实用建议

```
□ 对AI生成的事实性信息保持"默认怀疑"态度
□ 了解AI的训练数据截止日期
□ 重要信息使用多个独立来源交叉核实
□ 学习信息素养：区分事实、观点、分析
□ 记住：你有责任验证，不是AI有责任准确
```

## 第五课：版权与创作

### AI能创作吗？

**技术角度：** AI可以生成看起来像创作的东西。

**哲学角度：** 创作需要意图、体验、情感——AI没有这些。

### 版权争议的核心问题

```
1. 训练数据有版权吗？（用有版权的书/图片训练AI）
2. AI生成的内容有版权吗？
3. 谁拥有AI生成内容的版权？
4. 用AI辅助创作，版权归谁？
```

### 当前争议焦点

| 立场 | 观点 |
|------|------|
| 创作者 | AI在窃取我的劳动成果，生成的内容与我的作品太像 |
| AI公司 | AI学习的是模式，不是具体作品，这和人类学习一样 |
| 法律界 | 现有版权法没有明确界定，亟待更新 |
| 实用派 | 等法律搞清楚黄花菜都凉了，先用起来再说 |

### AI时代创作者的生存策略

1. **不要和AI比产量** → AI可以无限产出，你比不了
2. **但你可以比独特性** → 你的个人经历、情感、视角是AI无法复制的
3. **把AI当作工具** → 用AI放大你的创意，不是替代你的创意
4. **建立个人品牌** → 人们因为信任你而购买，不是因为某个作品
